Section 02

State Machine — Storyboard States

النظام يعمل بنظام حالات (State Machine) — كل حالة تتحكم بما يظهر على الشاشة وكيف يتفاعل المستخدم

stateDiagram-v2
    [*] --> EMPTY

    EMPTY --> PROCESSING: Click Create Storyboard
    PROCESSING --> ACTIVE: Scenes received
    PROCESSING --> EMPTY: API Error
    ACTIVE --> EMPTY: Click Reset

    state ACTIVE {
        IDLE2: IDLE
        CREATE2: CREATE
        EDIT2: EDIT
        
        [*] --> IDLE2
        IDLE2 --> CREATE2: handleCreateStoryboard
        CREATE2 --> IDLE2: Done
        IDLE2 --> EDIT2: handleEditStoryboard
        EDIT2 --> IDLE2: Done
    }
الشكل 2: مخطط الحالات — Status يتحكم بالعرض و Mode يتحكم بقفل التفاعل

What Each Status Renders

🔴 EMPTY

Textarea input
+ Create Storyboard button
+ Welcome overlay on canvas

🟡 PROCESSING

Spinner animation
+ "Generating Storyboard..."
+ Inputs disabled

🟢 ACTIVE

Scene list cards
+ Inline edit textarea
+ Full timeline active

Section 03

Create Storyboard — Sequence Flow

تسلسل عملية إنشاء الـ Storyboard من لحظة كتابة النص وحتى ظهور المشاهد الخمسة على التايملاين

sequenceDiagram
    actor User as 👤 User
    participant FE as 🖥️ Frontend
    participant BE as ⚙️ Backend
    participant AI as 🤖 Gemini AI

    User->>FE: Types script text
    User->>FE: Clicks "Create Storyboard"
    
    activate FE
    FE->>FE: status → PROCESSING
    FE->>FE: Show skeleton loader ⏳
    
    FE->>BE: POST /api/pipeline/create
    activate BE
    
    BE->>AI: Gemini 2.5 Flash + JSON Schema
    activate AI
    AI-->>BE: Array of 5 Scenes
    deactivate AI
    
    alt Success
        BE-->>FE: { scenes: [5 scenes] }
    else Failure
        BE->>BE: Paragraph split fallback
        BE-->>FE: { scenes: [5 fallback] }
    end
    deactivate BE
    
    FE->>FE: Map scenes → timeline elements
    FE->>FE: status → ACTIVE ✅
    deactivate FE
الشكل 3: تسلسل إنشاء Storyboard — يشمل المسار الناجح والمسار البديل (fallback)
💡
Gemini JSON Schema: The API uses responseMimeType: "application/json" with a strict schema forcing exactly scene_number, text, and duration_seconds per scene.
Section 04

Edit Scene — Sequence Flow

تسلسل عملية تعديل مشهد واحد — التعديل يعيد حساب المدة تلقائياً ويعدل التايملاين بالكامل بشكل تسلسلي

sequenceDiagram
    actor User as 👤 User
    participant FE as 🖥️ Frontend
    participant BE as ⚙️ Backend

    User->>FE: Select Scene #3
    FE->>FE: Show inline editor
    
    User->>FE: Edit text + click "Edit"
    
    activate FE
    FE->>FE: mode → EDIT
    FE->>FE: Shimmer overlay ✨
    
    FE->>BE: PUT /api/pipeline/edit
    activate BE
    BE->>BE: wordCount / 3 → clamp [3,15]
    BE-->>FE: { scene_id, text, duration_seconds }
    deactivate BE
    
    FE->>FE: Merge scene update
    FE->>FE: Recalculate all start times
    FE->>FE: mode → IDLE ✅
    deactivate FE

Vercel Deployment Architecture

معمارية النشر على Vercel — طلبات الـ API تروح لـ Serverless Function والباقي للـ Static SPA

graph TB
    BROWSER["🌐 Browser"] --> ROUTER["Vercel Router\nvercel.json"]

⚠️
Hobby Tier Limit: Vercel hobby tier has a 10-second execution limit. The system uses Promise.race() with an 8-second timeout to ensure graceful fallback before hitting the hard limit.

Duration Calculation Formula

المعادلة المستخدمة لحساب مدة كل مشهد بناءً على عدد الكلمات في النص

// Duration formula used in /api/pipeline/edit const wordCount = text.split(/\s+/).length; const duration = Math.max(3, Math.min(15, Math.ceil(wordCount / 3))); // Rule: ~1 second per 3 words, clamped between 3s and 15s
Input Text Word Count Raw Calc Final Duration
"Hello world"2ceil(0.67) = 13s (min clamp)
"The quick brown fox jumps"5ceil(1.67) = 23s (min clamp)
~15 word paragraph15ceil(5) = 55s
~30 word paragraph30ceil(10) = 1010s
~60 word paragraph60ceil(20) = 2015s (max clamp)
